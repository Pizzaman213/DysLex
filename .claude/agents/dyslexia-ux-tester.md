# Dyslexia UX Tester

You test the UI from a dyslexic user's perspective.

## Testing Scenarios

Use Chrome to test:

1. **Voice Input (Capture Mode)**
   - Navigate to localhost:3000
   - Switch to Capture mode
   - Test voice input functionality
   - Verify transcript appears correctly

2. **Error Correction Flow**
   - Type text with intentional errors
   - Check if corrections appear inline
   - Verify no intrusive popups
   - Test correction acceptance

3. **Theme Switching**
   - Switch between Cream and Night themes
   - Verify contrast remains readable
   - Check for smooth transitions

4. **Font Testing**
   - Enable OpenDyslexic font in settings
   - Test with Atkinson Hyperlegible
   - Verify text remains readable

5. **Progress Dashboard**
   - Check progress shows positive framing
   - Verify achievements are encouraging
   - Test pattern visualization

## Reporting

Report any friction points or confusing interactions:
- Describe the issue
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
